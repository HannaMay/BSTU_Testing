Case № 1: Ввод несуществующего места назначения
Шаги:
1.	Переход на главную страницу(booking.com)
2.	В поле поиска «Куда вы хотите поехатья?» ввод случайной последовательности символов
Ожидаемый результат:
	Сброс введенной последовательности (либо предложение частично совпадающих названий домов/отелей).
-------------------------------------------------------------

Case № 2: Выбор даты заезда из прошедшего времени или текущим днем
Шаги:
1.	Переход на главную страницу
2.	В поле поиска «Дата заезда» выбор даты из прошедшего времени или текущей даты
Ожидаемый результат:
	Прошедшие даты заблокированы для выбора пользователем, текущие даты ограничены временем работы отеля(дома).
-------------------------------------------------------------

Case № 3: Выбор даты отъезда из прошедшего времени или даты заезда
Шаги:
1.	Переход на главную страницу
2.	В поле поиска «Дата отъезда» выбор даты из прошедшего времени или даты заезда
Ожидаемый результат:
	Прошедшие даты, а так же даты прибытия в отель/дом заблокированы для выбора пользователем (минимум 1 ночь).
-------------------------------------------------------------

Case № 4: Выбор количества взрослых гостей меньше одного
Шаги:
1.	Переход на главную страницу
2.	В поле поиска «1 Взрослый - без детей - 1 номер» выбор 0 взрослых гостей кнопкой «-»
Ожидаемый результат:
	Количество взрослых гостей меньше одного заблокированы для выбора пользователем.

Case № 9: Выбор параметров поиска без указания даты отбытия
Шаги:
1.	Переход на главную страницу
2.	В поле «Укажите город, отель или место» выбор города» (Лондон)
3.	В поле «Заездз - Отъезд »
4.	В поле поиска «2 Взрослых ...» выбор двух взрослых (количество по умолчанию)
5.	Нажатие кнопки Проверить цены»
Ожидаемый результат:
	Автоматическая вызов календаря для уточнения дат прибытия-отбытия.

Case № 10: Выбор параметров поиска без указания даты прибытия
Шаги:
1.	Переход на главную страницу
2.	В поле «Куда вы ххотите поехать (Милан) и указываем даты +неделя ( Вместо кнопки "проверить цены" оттображается ценв за 1 ночь)
3.	Выбираем отель (Holiday Inn Milan Garibaldi Station) в списке и нажимаем "Выбрать номер"(BYN 324 + налоги и сборы (BYN 23) 
4.	В описании смотрим цены на 1 номер (BYN 325 + налоги и сборы (BYN 22)) суммарная стоимость = 347

Ожидаемый результат:
	Цена за 1 номер в списке отелей равна цене в описании отеля при выборе номера.

Case № 11: Выбор параметров поиска без указания даты прибытия
Шаги:
1.	Переход на главную страницу
2.	В поле «Куда вы ххотите поехать (Милан) и указываем даты +неделя ( Вместо кнопки "проверить цены" оттображается ценв за 1 ночь)
